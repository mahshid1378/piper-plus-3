## 概要 / Overview

piper-plus は 6 言語 (JA/EN/ZH/ES/FR/PT) に対応するマルチリンガル TTS モデルを提供しています。
コミュニティメンバーは、6 言語ベースモデルからファインチューニングしたモデルを投稿できます。
投稿されたモデルは `piper_plus_voices.json` に登録され、全ての piper-plus 実装 (Python / Rust / C# / Go / npm) から `--model <name>` で利用可能になります。

piper-plus provides multilingual TTS models supporting 6 languages (JA/EN/ZH/ES/FR/PT).
Community members can submit models fine-tuned from the 6-language base model.
Accepted models are registered in `piper_plus_voices.json` and become available across all piper-plus implementations via `--model <name>`.

## 前提条件 / Prerequisites

モデルを投稿するには、以下の条件を満たす必要があります。

| 条件 | 詳細 |
|------|------|
| ベースモデル | piper-plus 6 言語ベースモデル ([ayousanz/piper-plus-base](https://huggingface.co/ayousanz/piper-plus-base)) からファインチューニングしたもの |
| エクスポート形式 | ONNX (FP16) — `export_onnx.py` でエクスポート済み |
| 品質チェック | 全 6 言語で推論テスト済み |
| ライセンス | MIT 互換のライセンス (後述) |
| HuggingFace | モデルを HuggingFace にアップロード済み |

## データセット要件 / Dataset Requirements

### 発話数

| 用途 | 最小発話数 | 推奨 |
|------|-----------|------|
| ファインチューニング | 100 | 1,000+ |
| 事前学習 (新規ベースモデル) | 10,000+ | 50,000+ |

### 音声仕様

| 項目 | 値 |
|------|-----|
| サンプリングレート | 22,050 Hz |
| チャンネル | モノラル (mono) |
| フォーマット | WAV (16-bit PCM) |
| SNR | > 15 dB |
| 発話長 | 1 -- 15 秒 (推奨) |

### データセット構成

LJSpeech 形式 (フラットディレクトリ + `metadata.csv`) を使用してください。

```
dataset/
  wavs/
    utterance_001.wav
    utterance_002.wav
    ...
  metadata.csv          # format: id|text|normalized_text
  config.json           # generated by prepare_*_dataset.py
  phonemes.jsonl        # generated by preprocessing
  phoneme_ids.jsonl     # generated by preprocessing
```

## 品質チェックリスト / Quality Checklist

投稿前に、以下の全項目を確認してください。

- [ ] **6 言語推論テスト完了** -- 以下のテスト文で各言語の音声を生成し、品質を確認
  - JA: 「こんにちは、今日は良い天気ですね。」
  - EN: "Hello, how are you today?"
  - ZH: "你好，今天天气很好。"
  - ES: "Hola, como estas hoy?"
  - FR: "Bonjour, comment allez-vous?"
  - PT: "Ola, como voce esta hoje?"
- [ ] **音声長が妥当** -- テスト文で 1 -- 10 秒の範囲
- [ ] **FP16 ONNX エクスポート済み** -- `export_onnx.py` でエクスポート (デフォルトで FP16)
- [ ] **config.json の `num_symbols` が 173** -- 6 言語ベースモデルと一致
- [ ] **Model Card (README.md) 作成済み** -- HuggingFace リポジトリに配置
- [ ] **md5_digest 計算済み** -- ONNX ファイルと config.json の MD5 ハッシュを計算

### md5_digest の計算方法

```bash
# macOS
md5 -q model.onnx
md5 -q config.json

# Linux
md5sum model.onnx
md5sum config.json
```

## ファインチューニング手順 / Fine-tuning Steps

piper-plus の 6 言語ベースモデル (571 話者, 75 epoch) からのシングルスピーカーファインチューニング手順です。

### 1. データセット準備

```bash
# 前処理ツールでデータセットを作成
uv run python -m piper_train.tools.prepare_multilingual_dataset \
  --input-dir /path/to/your/wavs \
  --output-dir /path/to/dataset
```

### 2. ファインチューニング実行

以下は推奨パラメータです。`CONTRIBUTING.md` の Template B を参照してください。

```bash
uv run python -m piper_train \
  --dataset-dir /path/to/dataset \
  --prosody-dim 16 \
  --accelerator gpu --devices 1 --precision 32-true \
  --max_epochs 500 --batch-size 4 --samples-per-speaker 4 \
  --checkpoint-epochs 50 --quality medium \
  --base_lr 2e-5 --disable_auto_lr_scaling \
  --ema-decay 0.9995 \
  --max-phoneme-ids 400 \
  --no-wavlm \
  --val-every-n-epochs 50 \
  --audio-log-epochs 50 \
  --resume-from-multispeaker-checkpoint /path/to/base-model-checkpoint.ckpt \
  --default_root_dir /path/to/output
```

### パラメータガイド

| パラメータ | 推奨値 | 説明 |
|-----------|-------|------|
| `--base_lr` | 2e-5 | catastrophic forgetting 防止のため事前学習の 1/10 |
| `--max_epochs` | 500 | 100 発話なら 500 epoch で ~12,500 gradient steps |
| `--batch-size` | 4 | 小データセット向け |
| `--resume-from-multispeaker-checkpoint` | (必須) | emb_g 除去 + emb_lang 補正 + freeze-dp を自動実行 |
| `--no-wavlm` | (推奨) | V100 では学習速度優先 |
| `--precision 32-true` | (推奨) | V100 では FP16 mixed は backward pass が遅くなる |

### 3. 学習の確認

- WandB で loss カーブを確認
- `--audio-log-epochs 50` で定期的に音声サンプルを確認
- 音声が「ピー」音になる場合は `--base_lr` を下げる

## ONNX エクスポート手順 / ONNX Export Steps

学習完了後、チェックポイントを ONNX 形式にエクスポートします。

```bash
# 標準エクスポート (EMA + FP16、推奨)
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.export_onnx \
  /path/to/checkpoints/epoch=499-step=22000.ckpt \
  /path/to/output/model-name.onnx
```

**重要:**
- デフォルトで FP16 変換が適用されます (モデルサイズ ~50% 削減)
- シングルスピーカー多言語モデルでは `--unify-emb-lang` が自動有効化され、声質が統一されます
- `CUDA_VISIBLE_DEVICES=""` で CPU モードを使用してください

### エクスポートオプション

| オプション | デフォルト | 説明 |
|-----------|----------|------|
| `--no-fp16` | - | FP16 変換を無効化 |
| `--no-stochastic` | - | deterministic エクスポート (デバッグ用) |
| `--unify-emb-lang` / `--no-unify-emb-lang` | auto | emb_lang 統一の手動制御 |
| `--simplify` | - | ONNX simplification を適用 |

### エクスポート後の確認

```bash
# 推論テスト
CUDA_VISIBLE_DEVICES="" uv run python -m piper_train.infer_onnx \
  --model /path/to/model-name.onnx \
  --config /path/to/dataset/config.json \
  --output-dir /path/to/output \
  --text "こんにちは、今日は良い天気ですね。" \
  --language ja-en-zh-es-fr-pt --speaker-id 0
```

## voices.json エントリ / voices.json Entry

投稿が承認されると、`src/cpp/piper_plus_voices.json` にエントリが追加されます。
Issue 投稿時に以下のテンプレートを記入してください。

### エントリ形式

```json
{
    "<language_code>-<model_name>-<quality>": {
        "key": "<language_code>-<model_name>-<quality>",
        "name": "<model_name>",
        "language": {
            "code": "<language_code>",
            "family": "<lang>",
            "region": "<REGION>",
            "name_native": "<native_name>",
            "name_english": "<english_name>",
            "country_english": "<country>"
        },
        "quality": "medium",
        "num_speakers": 1,
        "speaker_id_map": {},
        "source": "community",
        "repo": "<huggingface_username>/<repo_name>",
        "files": {
            "<model_filename>.onnx": {
                "size_bytes": 0,
                "md5_digest": "<md5_hash>"
            },
            "config.json": {
                "size_bytes": 0,
                "md5_digest": "<md5_hash>"
            }
        },
        "aliases": ["<alias1>", "<alias2>"],
        "description": "<Short description of the model>"
    }
}
```

### 言語コード一覧

| 言語 | code | family | region | name_native | name_english | country_english |
|------|------|--------|--------|-------------|--------------|-----------------|
| 日本語 | ja_JP | ja | JP | 日本語 | Japanese | Japan |
| 英語 | en_US | en | US | English | English | United States |
| 中国語 | zh_CN | zh | CN | 中文 | Chinese | China |
| スペイン語 | es_ES | es | ES | Espanol | Spanish | Spain |
| フランス語 | fr_FR | fr | FR | Francais | French | France |
| ポルトガル語 | pt_BR | pt | BR | Portugues | Portuguese | Brazil |

### 命名規則

- キー: `<language_code>-<model_name>-<quality>` (例: `ja_JP-my-model-medium`)
- モデル名: 英小文字 + ハイフン区切り (例: `my-model`)
- エイリアス: 短縮名を含める (例: `["my-model", "ja-my-model"]`)
- source: コミュニティ投稿は `"community"`

## 投稿プロセス / Submission Process

### Step 1: HuggingFace にモデルをアップロード

1. [HuggingFace](https://huggingface.co) にアカウントを作成
2. 新しいリポジトリを作成 (例: `username/piper-plus-my-model`)
3. 以下のファイルをアップロード:
   - `<model-name>.onnx` -- FP16 ONNX モデル
   - `config.json` -- モデル設定ファイル
   - `README.md` -- Model Card (後述)

### Step 2: Model Card (README.md) を作成

HuggingFace リポジトリの README.md に以下の情報を含めてください。

```markdown
---
language:
  - ja
  - en
  - zh
  - es
  - fr
  - pt
license: cc-by-4.0
tags:
  - piper-plus
  - tts
  - onnx
---

# Model Name

## Description
(モデルの説明)

## Training
- Base model: piper-plus 6-language base (ayousanz/piper-plus-base)
- Dataset: (データセット名, 発話数)
- Epochs: (エポック数)
- Fine-tuning parameters: (主要パラメータ)

## License
- Model: (モデルのライセンス)
- Training data: (データセットのライセンス)

## Usage

```bash
# Rust
piper-plus --model <model-name> --text "こんにちは"

# C#
piper-plus --model <model-name> --text "こんにちは"
```
```

### Step 3: GitHub Issue で投稿

1. [Model Submission](https://github.com/ayutaz/piper-plus/issues/new?template=model-submission.yml) テンプレートを使用して Issue を作成
2. 全ての必須項目を記入
3. 品質チェックリストの全項目にチェック

### Step 4: レビューとマージ

1. メンテナーが以下を確認:
   - ライセンスの互換性
   - 音声品質 (サンプルを聴取)
   - config.json の整合性
   - HuggingFace リポジトリの構成
2. 承認後、メンテナーが `piper_plus_voices.json` を更新する PR を作成
3. PR マージ後、全 piper-plus 実装から `--model <name>` で利用可能に

## FAQ

### Q: ベースモデル以外からのファインチューニングは投稿できますか?

A: 現時点では piper-plus 6 言語ベースモデルからのファインチューニングのみ受け付けています。
これは `num_symbols = 173` やシンボルマッピングの互換性を保証するためです。

### Q: 多話者モデルは投稿できますか?

A: はい。`num_speakers` と `speaker_id_map` を正しく設定してください。
ただし、emb_lang の統一 (`--unify-emb-lang`) はシングルスピーカーモデル専用のため、多話者モデルでは使用しないでください。

### Q: GPU がありません。ファインチューニングできますか?

A: CPU でも可能ですが非常に時間がかかります。
Google Colab (GPU ランタイム) や、GPU を提供するクラウドサービスの利用を推奨します。

### Q: 6 言語以外の言語 (例: 韓国語, スウェーデン語) のモデルは?

A: G2P コードは 8 言語 (JA/EN/ZH/KO/ES/FR/PT/SV) に対応していますが、ベースモデルは 6 言語で学習されています。
KO/SV を含むモデルは、新しいベースモデルの学習が必要になるため、現時点では受け付けていません。

### Q: 投稿から公開までどのくらいかかりますか?

A: 通常 1 -- 2 週間です。レビューの状況により前後します。

### Q: Model Card は英語で書くべきですか?

A: 日本語でも英語でも構いません。両方併記を推奨します。

### Q: md5_digest は必須ですか?

A: はい。ダウンロード時の整合性チェックに使用されます。
ONNX ファイルと config.json の両方の MD5 ハッシュを計算してください。
