# v1.11.0 → v1.12.0 マイグレーションガイド

v1.12.0 には複数の Breaking change が含まれます。このガイドはユーザー種別ごとに必要な対応をまとめたものです。

> 完全な変更リストは [CHANGELOG.md](../../CHANGELOG.md) の `[1.12.0]` セクションを参照。

## TL;DR (種別ごとの最短対応)

| あなたが | 必要な対応 |
|---------|----------|
| **推論のみ利用** (PyPI/NuGet/crates.io/npm 経由) | `pip install -U piper-plus` 等で入れ替えるだけ。既存 ONNX モデルは引き続き動作 |
| **HiFi-GAN ckpt から学習継続中** | **学習継続不可**。`piper-plus-base` (MB-iSTFT) から再ファインチューニング必要 |
| **Flask `[http]` extras 利用** | `pip install -U "piper-plus[http]"` で FastAPI/uvicorn が自動 install。エンドポイント URL は互換 |
| **Unity UPM 利用** | `ayutaz/uPiper` 別 repo に移行済み。本リポジトリの `unity/` は削除 |
| **`PiperVoice.phonemize()` を直接呼んでいる** | 戻り値が複数文で N 要素返るようになった。`[0]` 固定アクセスを書き換え |

---

## 1. 推論ユーザー (Breaking なし)

各レジストリの新バージョンに更新してください。

```bash
# Python (PyPI)
pip install -U piper-plus

# C# (.NET 10 が必要)
dotnet tool update -g PiperPlus.Cli

# Rust
cargo install piper-plus-cli --force

# npm (Browser WASM)
npm install piper-plus@latest
```

既存の v1.11.0 ONNX モデル (HiFi-GAN ベースを含む) は **推論側で引き続き動作** します。Decoder が変わっても ONNX 出力テンソル形状 `[B, 1, T]` は維持されているためです (CHANGELOG L16)。

---

## 2. 学習ユーザー (HiFi-GAN ckpt は破棄要)

v1.12.0 で `Generator` (HiFi-GAN) クラスが削除され、Decoder は `MBiSTFTGenerator` に統一されました。

### 影響
- v1.11.0 以前の `.ckpt` で `--resume_from_checkpoint` または `--resume-from-multispeaker-checkpoint` を実行すると、`model_g.dec.*` の重み構造が一致せず、`unexpected keys` のエラーまたは警告で学習継続できません。
- `--mb-istft` フラグも廃止 (常時有効) されたため、旧コマンドラインから削除してください。

### 移行手順
1. **MB-iSTFT 対応の base モデルから再ファインチューニング**:
   ```bash
   # 6 言語 multispeaker base
   wget https://huggingface.co/ayousanz/piper-plus-base/resolve/main/model.ckpt \
     -O /data/piper/piper-plus-base.ckpt
   ```
2. CLAUDE.md の「Template B: シングルスピーカー ファインチューニング」コマンドで `--resume-from-multispeaker-checkpoint /data/piper/piper-plus-base.ckpt` を指定して再 FT を開始。
3. `--mb-istft` を含む既存スクリプトは引数を削除。

### 学習設定の差分
| 項目 | v1.11 (HiFi-GAN) | v1.12 (MB-iSTFT) |
|------|----------------|------------------|
| `--mb-istft` フラグ | 任意 | 廃止 (常時有効) |
| ckpt 互換性 | v1.11 同士で互換 | v1.11 ckpt 不可 |
| Decoder 計算量 | 168.2 ms / 100 phoneme (CPU p50) | 76.2 ms (~2.21x 高速化) |

## 3. HTTP Server ユーザー (Flask → FastAPI)

`pip install piper-plus[http]` の依存が変わりました。

### 影響
- 旧: `flask>=3,<4`
- 新: `fastapi>=0.110,<1`, `uvicorn[standard]>=0.27,<1`

### 移行手順
- `pip install -U "piper-plus[http]"` で自動切り替え。
- エンドポイント URL (`/api/tts`, `/api/phoneme-timing` 等) は **互換**。
- 起動コマンドが `python -m piper.http_server` のままなら変更不要。
- カスタム middleware を独自に注入していた場合は FastAPI 構文に書き換え必要。
- `?streaming=true` クエリで `StreamingResponse` による真のチャンク配信に対応 (PR #361)。

詳細: `src/python_run/README_http.md`

## 4. `PiperVoice.phonemize()` 直接利用ユーザー

`phonemize()` の戻り値**型** `list[list[str]]` は変わらないが、**意味論が変わった**ので呼び出し側コードに影響します。

### 影響例

```python
# v1.11.0 までは常に 1 要素
phonemes = voice.phonemize("こんにちは。さようなら。")
# → [["k", "o", "n", "n", "i", "ch", "i", "w", "a", "。", "s", "a", "y", "o", "u", "n", "a", "r", "a", "。"]]
ids = voice.phonemes_to_ids(phonemes[0])  # ✅ 動く

# v1.12.0 以降は文ごとに分割
phonemes = voice.phonemize("こんにちは。さようなら。")
# → [["k", "o", "n", "n", "i", "ch", "i", "w", "a", "。"],
#    ["s", "a", "y", "o", "u", "n", "a", "r", "a", "。"]]
ids = voice.phonemes_to_ids(phonemes[0])  # ⚠ 1 文目しか処理されない
```

### 移行手順
全文をまとめて処理したい場合:

```python
# 推奨: 文ごとに iterate
for sentence_phonemes in voice.phonemize(text):
    ids = voice.phonemes_to_ids(sentence_phonemes)
    # ... 推論
```

または、`synthesize()` / `synthesize_stream_raw()` / `synthesize_with_timing()` を使えば内部で iterate されるので変更不要です (これらの API は後方互換)。

SSML 入力 (`<speak>...`) は v1.11/v1.12 とも単一ユニット扱いです。

## 6. その他の細かい変更

### HTS voice 依存削除 (#342)
piper-plus の Python パイプラインから HTS voice (`*.htsvoice`) 依存を完全除去。OpenJTalk + pyopenjtalk-plus の純粋パスに統一。HTS voice ファイルを引数指定しているスクリプトがあれば削除。

### `pyopenjtalk-plus>=0.4` 要件
`pyopenjtalk-plus` は v0.4 以降が必要。古い `pyopenjtalk` (r9y9 オリジナル) からの移行は本ガイドの対象外 (v1.7.0 で既に切替済み)。

### npm: Node.js 24 LTS が必要
`piper-plus@0.6.0` は Node.js 24 以上を要求 (旧: 18 以上)。

### .NET 10 LTS が必要
`PiperPlus.Cli` `0.3.0` および `PiperPlus.Core` `0.3.0` は `.NET 10` ランタイムが必要 (旧: .NET 8/9)。

## トラブルシューティング

### Q. v1.11 の ckpt から resume したら "unexpected keys" 警告が出る
A. HiFi-GAN ckpt は学習継続不可です。`piper-plus-base` (MB-iSTFT) から再 FT してください (上記「2. 学習ユーザー」)。

### Q. `pip install piper-plus[http]` で fastapi が入らない
A. setuptools のキャッシュを更新: `pip install --upgrade --force-reinstall "piper-plus[http]"`

### Q. `piper-plus@0.6.0` を npm でインストールできない
A. 公開順序の関係で `@piper-plus/g2p@0.4.0` が先に publish されている必要があります。`npm cache clean --force && npm install piper-plus@latest` をお試しください。

### Q. Unity UPM のパッケージが見つからない
A. リポジトリが [ayutaz/uPiper](https://github.com/ayutaz/uPiper) に移行しました。
